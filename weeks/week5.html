Son distintas capas las que se han revisado hasta el momento, en esta semana revisaremos un poco más acerca de la capa de máquinas virtuales y la capa de S.O



DaaS

DaaS o Desktop as a Service es una solución de software que permite una conexión a un escritorio virtual por medio de una conexión por internet.

Citrix DaaS

Se pueden tener distintos usuarios a un mismo recurso. Mediante un servidor de perfiles, se guardan los archivo o la información de manera separada por usuarios.



Los servicios de DaaS por medio de los balanceadores de carga permiten entrar por distintas formas

  • User Layer: Esta capa describe el ambiente para usuarios finales y dispositivos finales que son usualmente conectados a los recursos:
    • Extenal User: Accenden al Workspace de Citrix para obtener acceso al Virtual Desktop hosteado en Azure
    • Intenal User: Cuando están en la oficina, acceden al pc físico. Cuando trabajan de manera remota Acceden al workspace de Citrix para obtener oacceeso a su pc físico
  • Acces Layer: Esta capa describe los detalles de los ambientes internos y externos de Citrix
    • Citrix Workspace: Es una solución de workspace digital que permite deliver acceso seguro a la información applicaciones, y otro contenido relevante a la persona con dicho rol
    • Gateway Service: Es un servicio de cloud que provee remoto acceso seguro con Identity and Access Management IdAM capabilities, entregando una experiencia SaaS, apps virtuales y escritorios virtuales
  • Resource Layer: Esta capa está encargada de gestionar los recursos de las applicaiones, escritorios e información:
    • Remote PC Access: Un tradicional escritorio de Windows asignado a un usuario y puede ser accedido de manera local o remota
    • Azure Virtual Deskotp: Escritorio virtualizado de windows 10 con multisesiones para usuarios que puedan acceder de manera remota a sus applicaciones
  • Control Layer: Esta capa balancea las sesiones disponibles y garantiza que los roles puedan acceder a la información correspondiente

Cloud Computing

Es la entrega de servicios de computo, incluyendo servidores, bases de datos, storage, networking y analitica en internet. Cloud ofrece beneficios como agilidad, flexibilidad, scalabilidad y economia de escala, permitiendo deploy soluciones de manera rapida y economica.

Organizaciones de todo tipo, tamaño e industrias están utilizando cloud para una gran variedad de casos, tales como data backup, disaster recovery, email, virtual desktops, software developments and testing, big data analytics, y customer-facing web applications. Por ejemplo, en compañias de healthcare están utilizando desarrollo en la nube para personalizar tratamientos a pacientes.

Tipos de Cloud Computing

  • Infrastructure as a Service: Está conformada por recursos de computo altamente escalables, esto en entrega bajo demanda de la infraestructura de computo. Eso significa que todos los recursos, como sistemas operativos, storage y networking son ofrecidos o entregados por el clould provider.
  • Software as a Service: Es cuando se utiliza una aplicación completa ofrecida por un tercero o un sistema. Los usuarios pueden acceder a las aplicaciones bajo demanda por Internet, sin tener que descargar o mantener el software.
  • Platform as a Service: Este tipo de solución es usualmente utilizado por los desarrolladores de software, quienes están buscando concentrarse en el desarrollo más que en DeVops y en la administración. Es una opción para no tener que preocuparse por la parte de infraestructura.

Tipos de ambientes de Cloud

  • Cloud: Una aplicación en cloud es completamente entregada por el proveedor de servicios y todas las aplicaciones se corren sobre el cloud.
  • Hybrid: Es un deployment que conecta aplicaciones e infraestructura de una solución cloud con recursos existentes que no se encuentra en cloud. El método más común es cuando se quiere extender las capacidades de una infraestructura on-premise para garantizar que la compañia siga creciendo.
  • On-Premises: Son recursos entregados, utilizado virtualización, normalmente llamada nube privada. Con los servidores instalados de manera local.

Core Services AWS

  • EC2 - VMs: Se encarga de ofrecer servicios de computo de manera escalable utilizando AWS.
  • VPC- Networking: Se encarga de poner los recursos de AWS bajo una network virtual definida por nosotros. Esta network virtual ensambla una red tradicional que operariamos en nuestro propio data center, con los beneficios de la escalabilidad de la infraestructura en AWS
  • S3 - Object Storage: Es un servicio de almacenamiento de objectos que ofrece escalabilidad, availability, seguridad y performance. Usuarios pueden utilizar este servicios para guardar información como data lakes, websites, mobile applications, backup, etc.
  • IAM: AWS Identity and Acces Managament, es una servicio web que permite administrar de manera segura los recursos de AWS.
  • RDS - Databases: Es un servicio web para crear, operar y escalar bases de datos relacionales.
  • Lambda Functions as a service: Permite correr snippets de código de manera serverless con HA

Windows Server

Windows es un sistema operativo utilizado de manera global en máquinas de hogares y de igual forma utilizado en gran cantidad a nivel de máquinas virtuales. Una máquina virtual necesita de 4 componentes importantes

  • Memoria RAM
  • CPU
  • Storage
  • Tarjeta de Red

Existe también el concepto de memoria virtual que permite utilizar o asignar de manera virtual espacio del Storage para poder lograr que aplicaciones puedan tomar de manera virtual este recurso y poder asignarlo y poder funcionar de manera correcta.

Al acceder a información que este almacenada en el storage, la experiencia del usuario puede verse afectada debido a que la Memoria RAM está más optimizada en términos de lectura/escritura.

Para realizar una operación que se encuentre el Storage es necesario mover la pagina hacia la RAM y luego ser procesada por la CPU, este proceso es realizado por el S.O pero a más profundidad, el procesador es el encargado de realizar dichos cambios.

Cuando alguien comenta que tiene una máquina que está paginando, es debido a que el procesador está ocupado moviendo paginas entre la memoria RAM y el disco duro. Por tanto la paginación de memoria es el proceso de intercambio. En Windows es conocido como archivo de paginación mientras en Linux es llamado memoria Swap Debido al swapping o paginación de bloques
